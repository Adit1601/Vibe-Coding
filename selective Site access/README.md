# 🛡️ Selective Site Access - Chrome Extension

A powerful Chrome extension for focused browsing and content control. Block specific sites, allow exceptions, and use advanced pattern-based rules to customize your browsing experience.

## ✨ Features

- 🚫 **Domain Blocking**: Block entire websites while allowing specific pages
- 🎯 **Pattern-Based Blocking**: Advanced rules for blocking specific URL patterns
- 🔐 **Password Protection**: Secure access with encrypted password system
- ⏸️ **Temporary Pause**: Disable blocking for specified time periods
- 🎨 **Focus Mode**: Strict blocking without exceptions for distraction-free browsing
- 🔄 **Whitelist Mode**: Block everything except explicitly allowed sites
- 🌙 **Dark Theme**: Modern, easy-on-the-eyes interface

## 🚀 Quick Start

### Installation
1. Download or clone this repository
2. Open Chrome and navigate to `chrome://extensions/`
3. Enable "Developer mode" in the top right
4. Click "Load unpacked" and select the extension folder
5. The extension icon will appear in your Chrome toolbar

### First-Time Setup
1. Click the extension icon to open the popup
2. Set a secure password (a random one is generated by default)
3. Start adding domains or patterns to block

## 📖 How to Use

### Basic Domain Blocking
1. **Add a domain**: Enter a domain like `facebook.com` in the Pattern Rules section
2. **Add exceptions**: Use the allowlist to permit specific URLs from blocked domains
3. **Quick actions**: Use the quick action buttons for common blocking scenarios

### YouTube Shorts Blocking (Popular Use Case)
To block only YouTube Shorts while keeping the rest of YouTube accessible:

1. Click the "Block YouTube Shorts" quick action button, or
2. Manually add a pattern:
   - **Type**: Path
   - **Pattern**: `*/shorts/*`
   - **Description**: Block YouTube Shorts

This will block URLs like:
- ✅ `https://youtube.com/shorts/abc123` (BLOCKED)
- ✅ `https://m.youtube.com/shorts/xyz789` (BLOCKED)
- ❌ `https://youtube.com/watch?v=abc123` (ALLOWED)
- ❌ `https://youtube.com/` (ALLOWED)

### Advanced Pattern Types

#### 1. **Path Patterns**
- **Purpose**: Block specific sections of websites
- **Example**: `*/admin/*` blocks all admin pages
- **Syntax**: Use `*` as wildcards for any characters

#### 2. **Domain Patterns**
- **Purpose**: Block entire domains or subdomains
- **Example**: `facebook.com` blocks Facebook
- **Syntax**: Simple domain names

#### 3. **URL Patterns**
- **Purpose**: Block specific URLs
- **Example**: `https://example.com/specific-page`
- **Syntax**: Complete URLs

#### 4. **Regex Patterns** (Advanced)
- **Purpose**: Complex pattern matching
- **Example**: `.*\\.example\\.com/restricted.*`
- **Syntax**: Regular expressions (with safety validation)

### Extension Modes

#### Focus Mode
- **Purpose**: Strict blocking without exceptions
- **When to use**: During work or study sessions
- **Effect**: Disables allowlist URLs temporarily

#### Whitelist Mode
- **Purpose**: Block everything except allowed sites
- **When to use**: Parental controls or extreme focus
- **Effect**: Reverses the blocking logic

### Pause Feature
- **Temporary disable**: Pause blocking for 15 minutes to 8 hours
- **Automatic resume**: Extension automatically re-enables after pause period
- **Visual countdown**: See remaining pause time in the popup

## 🔧 Configuration

### Storage Keys
The extension uses Chrome's sync storage with these keys:
- `blockedDomains`: Array of blocked domain names
- `allowlistUrls`: Array of allowed URLs
- `patternRules`: Array of pattern-based blocking rules
- `pauseUntil`: Timestamp for pause end time

### Pattern Rule Format
```javascript
{
  type: 'path|domain|url|regex',
  pattern: 'pattern string',
  description: 'Human readable description',
  enabled: true/false
}
```

## 🚨 Troubleshooting

### Extension Not Working
1. **Check permissions**: Ensure extension has necessary permissions
2. **Reload extension**: Go to chrome://extensions/ and click reload
3. **Check console**: Look for errors in browser developer tools

### Patterns Not Matching
1. **Test pattern**: Use the pattern testing feature
2. **Check syntax**: Verify pattern format is correct
3. **Case sensitivity**: Patterns are case-insensitive

### Password Issues
1. **Forgot password**: No recovery option - extension will need to be reset
2. **Change password**: Use the change password feature in settings
3. **Default password**: Generated randomly on first install

### Performance Issues
1. **Too many patterns**: Limit to essential patterns only
2. **Complex regex**: Avoid overly complex regular expressions
3. **Browser restart**: Sometimes needed after major changes
